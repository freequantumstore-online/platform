# AGENTNAME — Extractor Agent

A **data extractor** heuristic agent on [FreeQuantumStore](https://freequantumstore.online).

Extractors pull structured data from unstructured input (HTML, text, email):
form fields, job details, prices, contact info, navigation links, metadata.

## How extractors work

1. **Locate**: Find relevant elements via tag patterns, attributes, selectors
2. **Classify**: Determine what each element is (email field, price, title, etc.)
3. **Extract**: Pull the value and build a CSS selector or path to reach it
4. **Score**: Assign confidence per extracted field

The heuristic code is generated by an LLM and evolved against test fixtures.
At runtime: zero model, zero tokens, sub-millisecond extraction.

## Output shape

Every extractor returns an array of found items:
```ts
{
  fields: Array<{
    role: string;         // what this field is
    value?: string;       // extracted value (if applicable)
    selector: string;     // CSS selector to find it in the DOM
    confidence: number;   // 0-1
    evidence: string[];   // why we classified it this way
  }>;
  metadata: Record<string, unknown>;  // agent-specific summary data
}
```

## Training

1. Add test HTML/text fixtures to `web/src/heuristic.test.ts`
2. Use the [Trainer](https://freequantumstore.online/a/trainer/) to evolve the heuristic
3. Iterate: add failing test cases, re-evolve, repeat until accuracy target

## Use cases

- **Form fields** — find email/password/name inputs for auto-fill
- **Job details** — extract title, company, salary, location from job pages
- **Prices** — find product price, currency, sale price
- **Navigation** — extract menu links, breadcrumbs, pagination
- **Contact info** — pull email, phone, address from any page
- **Metadata** — title, description, OG tags, structured data

Each saves ~800-2000 LLM tokens per page vs sending HTML to an API.
Especially valuable for high-volume automation (hundreds of pages/day).
